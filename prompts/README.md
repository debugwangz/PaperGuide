# PaperGuide 提示词

本目录包含 PaperGuide Agent 使用的所有提示词模板。

## 文件说明

| 文件 | 用途 |
|------|------|
| `system.md` | Agent 核心人设、逻辑链条规则、关键知识点识别 |
| `paper_analysis.md` | 论文分析和深度解读文档生成 |
| `knowledge_gap.md` | 检测逻辑断层和知识空白 |
| `image_types.md` | 图片分类和解读 |
| `md_update.md` | 判断是否需要更新 MD |

## 核心设计原则

### 1. 逻辑链条清晰

每引入一个新概念，必须先回答："为什么要讲这个？"

```
1. 我们想做什么？（目标）
2. 遇到什么问题？（困难）
3. 怎么解决？（引入新概念的动机）
4. 这个概念是什么？（定义）
5. 具体怎么用？（方法）
```

### 2. 从 Why 到 What 到 How

先讲为什么需要，再讲是什么，最后讲怎么做。

### 3. 关键知识点重点讲

- 核心概念：一句话定义 + 类比 + 为什么重要 + 正式定义
- 次要概念：一句话带过

### 4. 不要照搬论文

论文是给专家看的，我们的文档是给初学者看的。不是翻译论文，是重新组织知识。

## 变量说明

使用 `{variable}` 格式的占位符：

| 变量 | 说明 |
|------|------|
| `{title}` | 论文标题 |
| `{paper_content}` | 论文内容 |
| `{user_background}` | 用户背景描述 |
