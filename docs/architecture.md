# PaperGuide 架构设计

## 项目愿景

**让任何人都能读懂任何论文。**

学术论文往往假设读者具备大量背景知识，对新手极不友好。PaperGuide 通过 AI 填补这一鸿沟：
- 自动检测论文中的"知识断层"
- 通过网络搜索补充背景知识
- 用对话方式解答疑惑
- 生成适合读者水平的解读文档

## 核心需求

### 用户场景

1. **跨领域研究者** - 需要快速了解陌生领域的论文
2. **本科生/研究生** - 阅读第一篇论文，缺乏背景知识
3. **工程师** - 想了解最新研究但没时间深入

### 功能需求

| 需求 | 描述 | 状态 |
|------|------|------|
| PDF 解析 | 提取文本、章节结构、图表 | ✅ 完成 |
| arXiv 支持 | 通过链接/ID 自动下载 | ✅ 完成 |
| 论文分析 | 生成结构化解读文档 | ✅ 完成 |
| 对话问答 | 针对论文内容实时问答 | ✅ 完成 |
| 文档更新 | 对话中的知识更新到文档 | ✅ 完成 |
| 知识补充 | 搜索补充背景知识 | 🚧 基础完成 |
| 图片解读 | 解析论文中的图表 | 🚧 基础完成 |
| 多语言 | 支持中英文论文 | 📋 计划中 |
| 导出 | 导出为 PDF/Notion | 📋 计划中 |

## 数据流

### 1. 论文分析流程

```
用户上传 PDF / 输入 arXiv 链接
           │
           ▼
    ┌──────────────┐
    │  PDF Parser  │ ──→ 提取文本、章节、图片
    └──────────────┘
           │
           ▼
    ┌──────────────┐
    │  LLM 分析    │ ──→ 使用 paper_analysis.md 提示词
    └──────────────┘
           │
           ▼
    ┌──────────────┐
    │ 生成 Review  │ ──→ 结构化 Markdown 文档
    └──────────────┘
           │
           ▼
    保存到 outputs/{paper_name}/review.md
```

### 2. 对话问答流程

```
用户提问
    │
    ▼
┌────────────────────┐
│ 构建上下文         │ ──→ Review + 对话历史 + 问题
└────────────────────┘
    │
    ▼
┌────────────────────┐
│ LLM 生成回答       │
└────────────────────┘
    │
    ▼
┌────────────────────┐
│ 判断是否更新文档   │ ──→ 使用 md_update.md 提示词
└────────────────────┘
    │
    ▼
用户确认后更新 Review
```

## 模块详解

### PaperGuideAgent (core/agent.py)

核心协调者，负责：
- 加载论文 (PDF/arXiv)
- 调用 LLM 分析论文
- 管理会话状态
- 处理对话问答
- 更新 Review 文档

### Session 数据结构

```python
@dataclass
class Session:
    user_background: str        # 用户背景描述
    key_questions: list[str]    # 用户提过的问题
    clarified_concepts: list    # 已解释的概念
    pending_gaps: list[str]     # 待补充的知识空白
    conversation_history: list  # 对话历史
```

### LLM Client (core/llm_client.py)

统一的 LLM 调用接口，支持：
- OpenAI (GPT-4o 等)
- Claude (Sonnet/Opus)

通过 `LLM_PROVIDER` 环境变量切换。

### Search Module (core/search/)

网络搜索模块，用于补充背景知识：
- 抽象接口 `SearchProvider`
- 当前实现：Brave Search
- 可扩展：Tavily、Google 等

## 输出结构

每个论文分析会创建独立目录：

```
outputs/
└── {paper_name}/
    ├── review.md       # 论文解读文档
    ├── session.json    # 会话状态
    └── images/         # 提取的图片
```
